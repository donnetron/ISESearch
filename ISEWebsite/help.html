How to use Don's ISE Search tool

Purpose

The Irish Stock Exchange (ISE) includes notices published by all companies with securities listed in Ireland. For tax reasons, and because Ireland has a relatively cheap and streamlined process for listing on an EU regulated market, many of the listed debt securities we work on in SFG are Irish listed.

On the Irish Stock Exchange website, it is possible to view all of the notices published by a company with listed securities. Currently the database as it is presented is searchable by company name only. While somewhat useful, there are a number of 'annoyances' which mean that it can be difficult to find a notice about a particular subject (often a notice headline is misleading or too generic). It is also difficult to download multiple notices for printing without clicking on each one manually.

This website has downloaded every single notice published on the Irish Stock Exchange since January 2003 andindexed them based on their content in order to perform full word searches (as you would in Google) and get a better set of results. Alternatively you can search for a keyword, and then filter for a specific company name or by date using the 'Advanced' settings.

Finally you are able to download your results in a zip file in PDF or HTML format for printing.

You can read about how the site was made here.

How to use

Type in a keyword or company name to get all of the notices published that contain that word. There is an autocomplete function which pops up with suggestions. If no autocomplete suggestion comes up there are zero matches.

Searches are by default based on "SearchTerm1 OR Searchterm2". So if you search for "Morgan Citi" your results will show all notices with the word "Morgan" or the word "Citi" in them. If you want to find notices with both terms, search for "Morgan AND Citi".

From the results displayed, you can click on the link to view the notice directly on the ISE website. There is also a link to a 'cached' version of the notice stored locally in case the ISE website is down.

You can use the checkbox to select the notices you want to download or the checkbox at in the column heading to select the all results on a page, all results in total (if multiple pages), or none of the results from a search. Once you have at least one notice selected you can click the download button for a zip file or PDF containing them. If you download a zip file of notices, it is the cached version that is retrieved.

Limitations

Limited time, experience, and hardware have meant there are a few shortcuts on the website. It's not designed to be robust for mass use, and is more a proof of concept.

  • Updates daily only - the website works by scanning the notices published on the Irish Stock Exchange periodically and updating a database. I have set this to complete at the end of each day which means if looking for notices published on the same day they won't be found.
  • It's slow (especially downloading PDFs) - it runs on an old laptop in my broom cupboard which means that it won't handle many users at the same time. If downloading PDFs of your search results, the conversion can take a few minutes if there are a lot of files. It is quicker to save the HTML pages.
  • It might not be reliable - For the same reason that its slow, there is no telling when the laptop might conk out so it might go down some day.
  • You can only download 200 files at a time - this is a self imposed limitation that prevents someone from slowing the system down by downloading thousands of a files. If you needed to download more than 200 you could filter by a date range to do several batches.
  • Back-button doesn't work - If you read the about page you will see that the site makes use of AJAX to give fast search results without refreshing the page. For this reason you spend most of the time in a single page that updates the content dynamically. Without quite a lot of extra work it is difficult to implement correct use of the back-button on your browser to look at old searches.
  • No highlighting - some searches bring up results that you wouldn't think normally match. Usually the keyword is buried somewhere difficult to spot (but it really is there). There might be some way to automatically highlight matches on screen but no time to implement this. Sorry!
